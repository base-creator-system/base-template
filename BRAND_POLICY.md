# BASE System — Brand Policy

## 1. Purpose

This policy defines how the BASE name and identity may be used.

The goal is to protect the integrity of BASE as a system while allowing open use of the code.

---

## 2. What "BASE" Refers To

"BASE" refers specifically to:

* The original system architecture
* The creator-defined structure and philosophy
* The official version of the BASE System

---

## 3. Allowed Use

You MAY:

* State that your app is "built with BASE"
* Reference BASE as the system used to create your app
* Explain BASE as part of your workflow

Examples:

* "Built with BASE"
* "Powered by BASE"
* "Created using the BASE System"

---

## 4. Not Allowed

You MAY NOT:

* Rename your version as "BASE"
* Present your version as the official BASE system
* Claim ownership of BASE
* Represent BASE as a platform you operate
* Create confusion between your project and the original BASE system

---

## 5. Modified Versions

If you modify BASE:

* You must clearly indicate it is a modified version
* You must not present it as the original system
* You must not use BASE branding as the primary identity

Example:

✔ "Custom build based on BASE"
✘ "BASE Pro"
✘ "BASE Network"

---

## 6. Logos and Visual Identity

If logos are provided:

* They may only be used to reference the original BASE system
* They may not be altered or rebranded
* They may not be used as the identity of a separate product

---

## 7. No Platform Framing

BASE must not be described or positioned as:

* A platform
* A centralized service
* A network owned or controlled by any single operator

BASE is a system used independently by each creator.

---

## 8. Enforcement

Misuse of the BASE name or identity may result in:

* Revocation of permission to use branding
* Public clarification of misuse

---

## 9. Core Principle

You own what you build with BASE.

You do not own BASE itself.

BASE remains a system — not a platform.
