# BASE CANON — PUBLIC
Version: 1.0
Status: ACTIVE

## 0) Purpose
Base exists to give creators ownership-first digital homes they control.

Base enables creators to publish, connect with their audience, and earn
ethically without forced dependence on algorithmic platforms.

---

## 1) What Base Is
Base is a creator-owned Progressive Web App (PWA) system built on a stable core.

Each creator app:
- Shares a common system foundation
- Can look unique
- Retains ownership and autonomy
- Operates without algorithmic manipulation

Base is a system you can clone, customize within rules, and deploy.

---

## 2) What Base Is Not
Base is not:
- A social media platform
- An algorithmic feed
- An ad network
- A data-harvesting service
- A marketplace that owns creators

Base does not require:
- App store approval to be valid
- Platform dependency to function

---

## 3) System Model (4 Layers)
Base operates in four layers:

1. Founder Governance
2. Base Core Logic
3. Base UI System
4. Creator Apps (Pilots)

Creators build in Layer 4.
Layers 1–3 remain stable and protected.

---

## 4) Non-Negotiables
Base enforces the following principles:

- Creator ownership is preserved
- No forced extraction models
- Stability is favored over speed
- System rules are explicit and documented
- Silent changes to Base Core are not allowed

---

## 5) Locked vs Editable
Base separates change clearly:

### Locked
- Core system rules
- Navigation logic
- Base UI component behavior
- Security posture
- Versioning discipline

### Editable
- Branding
- Content
- Page composition using Base components
- Creator-specific sections

---

## 6) Monetization Doctrine
Base allows ethical monetization through:
- Optional infrastructure tools
- Premium features that do not remove ownership
- One-time build or deployment services

Base does not monetize through:
- Selling user data
- Forced ads
- Manipulative engagement systems
- Pay-or-lose-access models

---

## 7) Safety & Integrity
Base does not support systems built for:
- Exploitation or abuse
- Surveillance or malware
- Harassment or trafficking
- Hate or dehumanization

Access to Base may be refused if integrity is at risk.

---

## 8) Pilots
Pilots are controlled implementations used to validate Base.

- Early pilots proved viability
- Canon defines the rules
- Future pilots validate stability

Pilots do not redefine Base.
Canon does.

---

## 9) Change Control
Changes to Base Core require:
1. Documentation first
2. Explicit approval
3. Canon update
4. Then implementation

---

## 10) Final Public Statement
Base evolves by rule, not impulse.

If a build violates this Canon, it is not Base.
